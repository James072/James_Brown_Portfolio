Explanation
Provisio was the class project for CSD 460 Capstone Project course at Bellevue University. The class was divided up into teams of 5 and each team was required to produce their own version of a Provisio website, over a 12-week period, from the case study presented. The team I was assigned to was named "Charlie Team," comprised of the following students: Louis Thiemann(Team Lead), James Brown(Me), Kyle Emory, Abigail Decker and Andy Nguyen. We created a Discord channel in order to communicate and quickly share files, along with a Github repository to upload files and manage tasks on our Kanban board.
Project Evolution
According to the case study, Provisio was a hotel company with 3 locations, and it was left up to each team to pick 3 places to locate the properties. Charlie Team chose Omaha(NE), Grand Canyon(AZ), and Orlando(FL). We were instructed to create a landing page, a reservation page, a search existing reservations page, a locations page and a rewards program that allows customers to register on the website and earn reward points for hotel stays. Once we narrowed down the data that would be required, we developed a relational database using MySQL, then, over the following weeks, we individually and collaboratively created the different parts of the web site. As we all were assigned different tasks, my contribution consisted of the following:
  • Contributed to database design and implementation
  • Landing/Reservation page(front & back-end)
  • Search Existing Reservations page(front & back-end)
  • Locations page(front & back-end)
  • The "PROVISIO" logo
Challenges
The main challenge of this project was the communication between the team leader and myself. The tasks that I was assigned required me to produce both front-end and back-end code, which I was able to do. When my tasks were completed, I would turn the code over to the team lead for review. The team lead would review the code, approve it, and then implement it into the overall project. Upon viewing the end result, I would discover that the team lead had hard-coded all of the data that was supposed to be dynamically generated from the database. This was very frustrating because I was under the impression that the team would receive a low grade for not implementing the database functionality. After discussing this with the team lead, he admitted to me that he wasn't that familiar with back-end coding when it came to SQL and Java.

Although it was not a requirement, responsive design was not implemented. Overall, we still received an 'A' on the project.
