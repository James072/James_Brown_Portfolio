TechWorld is a mock ecommerce website that I created to demonstrate my full stack development skills. I utilized Java Programming's Spring Boot framework to create a backend API that serves product data to the frontend, which was rendered using Thymeleaf templates and Bootstrap for responsiveness. The website features a clean and modern design, with a focus on user experience and functionality.

The screen shot to the left shows the home / landing page of the site, which features a carousel of products from each category, as well as a navigation bar that allows user to browse each product category. The screen shot to the right shows a page of all existing products on the site, which is accessed by clicking the "Products" link on the left side of the navigation bar.

The screen shot to the left shows the middle portion of the page that shows all of the existing products on the site, while the screen shot to the right shows the "Servers" category page, showing an example of how each category page is structured, which is accessed by clicking the "Servers" link on the left side of the navigation bar.

The screen shot to the left shows a side panel extending from the left side of the screen that displays a list of manufacturers, which are accessed by clicking the "Find By Brand" button located directly above the displayed products on each product category page (see above screen shots), allowing a user to filter products by manufacturer. The screen shot to the right shows the "Contact Us" page containing a form that allows users to submit questions or comments about the site, which is accessed by clicking the "Contact Us" link located on the left side of the navigation bar.

The screen shot to the left shows the page that displays a single product when a user clicks the "View Details" button located at the bottom of each product display card, which contains a larger image of the product, as well as more detailed information about the product. There are also two buttons located below the product description allowing a user to either add the product to their shopping cart, or return to the home page. The screen shot to the right displays the shopping cart page, which appears once a user clicks the "Add To Cart" button. The user is shown a thumbnail image of the product they added, along with the price, the quantity, which can be adjusted by the user, the price subtotal, followed by a button that when clicked, will remove the product from the cart. Directly below the action button is a section displaying the total price of all products, and under that is a "Proceed To Checkout" button that directs the user to a checkout page and a "Continue Shopping" button that directs the user back to the Products page.

The screen shot to the left shows the top portion of the checkout page that contains a form that collects the user's shipping information, along with phone number and email address, and provides fields for entering payment information, with options for using credit/debit cards or ACH payments. The screen shot to the right shows the bottom portion of the checkout page displaying the order subtotal, tax, shipping cost and total price, which the user can review before clicking the "Complete Purchase" button directly above.

The screen shot to the left shows the order receipt page that appears after a user clicks the "Complete Purchase" button. Each receipt is given a unique transaction ID, and displays the order details, including the products purchased, the total price, and the shipping information. The screen shot to the right shows the account creation page that allows users to create an account on the site, which is accessed by clicking the "Sign Up" button located on the upper right side of the page, above the navigation bar. Users can create an account by entering their name, email address and creating a password, which allows them to save their shipping information for future purchases and view their order history.

The screen shot to the left shows the account login page that allows users to log in to their account by entering their email address and password, which is accessed by clicking the "Log In" button located on the upper right side of the page, above the navigation bar. The two screen shots to the right show the changes that occur to the navigation bar once a user logs in. When a regular user logs in, a "Order History" link appears on the left side of the navigation bar which allows a user to check their order history. When an admin user logs in, the "Order History" link appears, along with an "Admin Portal" link that leads to an admin dashboard where the admin can manage products, view orders and perform other administrative tasks, as shown in the screen shot on the upper right side. The screen shot on the lower right side shows the email address of the account that is logged in, appearing on the upper right side of the page, above the navigation bar, next to the "Log Out" button.

The screen shot to the left shows the main dashboard inside of the Admin Portal consisting of five buttons that lead to different administrative functions. The screen shot to the right shows the "Manage Inventory" page, which appears when the blue button labeled "Total Products" on the dashboard is clicked. This page displays a table consisting of product IDs, product names, thumbnail images of the products, categories, prices, stock quantities, and buttons to edit or delete each product. There is also a button at the top of the page that allows the admin to add new products to the inventory, which leads to a form where the admin can enter the new product's name, description, price, stock quantity, category and upload an image for the product.

The screen shot to the left shows the Order Ledger page that appears when the green button labeled "Total Orders" on the dashboard is clicked. This page displays a table of email addresses belonging to customers who have made purchases on the site, along with the transaction IDs of their orders, the total price of each order, and the date each order was placed. The screen shot to the right shows the Customer Directory page that appears when the gold button labeled "Registered Customers" on the dashboard is clicked. This page displays a table of all registered users on the site, along with their email addresses, names, account creation dates and account types (regular user or admin user).

The screen shot to the left shows the "Manage Categories" page that appears when the light-blue button labeled "Manage Categories" on the dashboard is clicked. This page displays the information stored in the categories table of the database, which consists of category IDs, category names, along with a component that allows the admin to add new categories. The screen shot to the right shows the "Manage Manufacturers" page that appears when the black button labeled "Manage Manufacturers" on the dashboard is clicked. This page displays the information stored in the manufacturers table of the database, which consists of manufacturer IDs, manufacturer names, along with a component that allows the admin to add new manufacturers.
