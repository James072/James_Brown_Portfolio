TechWorld
Explanation
TechWorld was my capstone project for INFO 2940 Computer Programming Capstone course at Metro Community College. The original use case that I created for this project was a company that sells new and used technology products by way of mail order who now needed to update to an order processing system that would allow customers to place orders online. The updated system would also include an administrative dashboard that would allow the company to manage inventory, view orders, and manage customer information.
Project Evolution
The original iteration of TechWorld was built using Java Swing, which is referred to as a GUI (Graphical User Interface) toolkit, but due to its inversion of control, comprehensive infrastructure, and design patterns, it is considered a legacy framework. My idea was to build the user interfaces with Swing, while the backend would be built with Java and MySQL. I started with 3 GUIs - a customer interface, a product interface, and a shipping interface. I created the database, implemented the backend code and connected the GUIs to the backend using JDBC (Java Database Connectivity). What I did not understand at the time was that Swing is not a web-based framework, so the GUIs that I created were not able to be deployed to the web. After some discussions with the Dean of IT, I was still able to earn my associates degree and graduate. I moved on to a 4-year university to pursue a bachelor's degree in Software Development where I took advanced courses in Java and web development and learned about Java Server Pages (JSP), which is a server-side technology that enables you to create dynamic web pages by embedding Java code in HTML. I revisited TechWorld and started rebuilding it with JSPs, but due to outside interference, my learning was limited and I ran into issues.

As a part of my continuous learning, I took the initiative to learn the Spring Boot framework, which is a powerful and widely used Java framework for building web applications. I once again revisited TechWorld and began to migrate the JSP version over to Spring Boot, which allowed me to create a more robust and scalable application. I implemented the backend with Spring Boot and connected it to a MySQL database using Spring Data JPA, Hibernate and Object Relational Mapping (ORM). For the frontend, I used Thymeleaf, which is a server-side Java template engine that allows you to create dynamic web pages. I also implemented Bootstrap for styling and responsiveness. Functionality also exists that allows for customer account registration and login which was configured using Spring Security. The current version of TechWorld is deployed on Railway.app, which is a cloud platform that allows you to easily deploy and manage your applications by establishing a connection to your GitHub repository. This version also has a customer-facing storefront, a shopping cart, payment processing, and an administrative dashboard that allows an administrator to manage the customer data and perform CRUD operations on the database.

The Downtown Diner
Explanation
The Downtown Diner was the final project for INFO 1311 Web Page Creation course at Metro Community College. In the case study, the diner had already been opened for years and now needed a website to advertise on the internet. The name of the diner, the owner's names and diner location was given in the instruction for the project. The rest was left up to the developer's interpretation. This was a static website in which the instructions called for a home page, a page explaining the diner's history, a page displaying menu specials, and a contact page that provides a form for customers to ask questions or leave comments.
Project Evolution
From the case study, Downtown Diner was represented as a diner that served comfort food with a down home appeal, so I wanted that representation to come through in the look of the site. When I thought "down-home", the look of wood grain came to mind. I scoured the internet for a wood panel image to use as the background for each page. Once I found that, I searched for images of single planks of wood and found some animated wood pieces that became the navigation buttons. I then used CSS to add borders that look like wooden picture frames around images and other elements. In 2023, after learning the Bootstrap framework, I revisited this project and restructured it to tighten up the responsiveness of the site. I added a JavaScript element to the Specials page that displays the daily specials based on the day of the week. I also added a page that allows a user to place an online order (the functionality is currently being implemented).

Provisio Hotels
Explanation
Provisio was the class project for CSD 460 Capstone Project course at Bellevue University. The class was divided up into teams of 5 and each team was required to produce their own version of a Provisio website, over a 12-week period, from the case study presented. The team I was assigned to was named "Charlie Team," comprised of the following students: Louis Thiemann(Team Lead), James Brown(Me), Kyle Emory, Abigail Decker and Andy Nguyen. We created a Discord channel in order to communicate and quickly share files, along with a Github repository to upload files and manage tasks on our Kanban board.
Project Evolution
According to the case study, Provisio was a hotel company with 3 locations, and it was left up to each team to pick 3 places to locate the properties. Charlie Team chose Omaha(NE), Grand Canyon(AZ), and Orlando(FL). We were instructed to create a landing page, a reservation page, a search existing reservations page, a locations page and a rewards program that allows customers to register on the website and earn reward points for hotel stays. Once we narrowed down the data that would be required, we developed a relational database using MySQL, then, over the following weeks, we individually and collaboratively created the different parts of the web site. As we all were assigned different tasks, my contribution consisted of the following:
  • Contributed to database design and implementation
  • Landing/Reservation page(front & back-end)
  • Search Existing Reservations page(front & back-end)
  • Locations page(front & back-end)
  • The "PROVISIO" logo
Challenges
The main challenge of this project was the communication between the team leader and myself. The tasks that I was assigned required me to produce both front-end and back-end code, which I was able to do. When my tasks were completed, I would turn the code over to the team lead for review. The team lead would review the code, approve it, and then implement it into the overall project. Upon viewing the end result, I would discover that the team lead had hard-coded all of the data that was supposed to be dynamically generated from the database. This was very frustrating because I was under the impression that the team would receive a low grade for not implementing the database functionality. After discussing this with the team lead, he admitted to me that he wasn't that familiar with back-end coding when it came to SQL and Java.

Another issue with the project was the lack of responsiveness of the site. I don't believe that any of the team members, including self, had any experience with a front-end framework at the time, and there were no media queries implemented in the CSS. Although we received an 'A', these issues bothered me

Dvyne Soulz, Inc.
Explanation
Dvyne Soulz, Incorporated is a non-profit organization whose base of operations is in Lincoln, Nebraska. Founded by Deon Jackson, Dvyne Soulz mission is to improve the condition of the community through cognitive programs. Having already been in operation, Deon approached me and explained that he needed a website for his organization to actually comply with requirements for certain funding.
Project Evolution
After the initial consultation with Mr. Jackson, I determined that what he needed was a website that would be strictly informational and contain mostly static content. Based on that meeting, I created a landing page which presents the logo, the name of the organization, and a few introductory words, followed by an "ENTER" button that takes a user into the site. Once inside, the Home page briefly presents concepts supported by the organization, along with images of youth that have previously participated in a Dvyne Soulz program. The About Dvyne Soulz page explains the mission, vision, and purpose of Dvyne Soulz and includes some words from the founder himself, along with images of Mr. Jackson and some of his own children. The T.R.A.I.T.S. page explains the main curriculum implemented at Dvyne Soulz, which is the T.R.A.I.T.S. (Trades. Resources. Affirmations. Identity. Talent. Self-Awareness.) program. The Donate page is a simple layout with a short paragraph asking users of the site to make a monetary donation in support of Dvyne Soulz. There is a donate button at the bottom of the paragraph that is linked to a third-party where the donation can be made.
